BRANCHS
    Ejecutar los siguiente comandos, equipo front y QA:

    git checkout -b ＜nueva-rama＞ ＜rama-existente＞
    ejemplo: git checkout -b front develop

Cada rol debe tener su proyecto base en el rol que le corresponda y en su subrama individual trabajar.

Luego crear otras subramas ya personales de cada uno dentro del rol que le corresponda.
ejemplo: git checkout -b elias backend
